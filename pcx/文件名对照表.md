# PCX 文件名对照表

MegaDesc 的运行时素材统一放在插件目录的 `pcx` 子目录，文件名统一使用 `bv_` 前缀。

## 命名限制

- 运行时素材由插件直接从插件目录 `pcx` 子目录读取，不走 HoMM3 原版资源管理器。
- 因此文件名不受 HoMM3 12 字节资源名缓冲限制。
- 当前配置字段 `BackgroundPcx` 使用 2K 堆缓冲，只写文件名，不需要写 `pcx\` 目录或完整路径。
- 按钮素材文件名目前在代码中固定写死，保持现有 `bv_` 前缀即可。

## 命名规则

```text
bv_<类型><编号/状态>.pcx
```

| 前缀 | 含义 | 尺寸 | 运行时使用 |
|------|------|------|------------|
| `bv_bg` | 背景图 | 298×487 | 是 |
| `bv_prev` | 效果预览图 | 298×487 | 否，仅预览 |
| `bv_frm` | 按钮金框 | 50×42 / 50×38 | 是 |
| `bv_ok` | 确认按钮图标 | 48×40 | 是 |
| `bv_sp` | 魔法书按钮图标 | 48×36 | 是 |
| `bv_ds` | 解雇按钮图标 | 48×36 | 是 |

按钮图标末尾 `0`-`3` 对应 DEF 帧号：

| 后缀 | 状态 | 说明 |
|------|------|------|
| `0` | normal | 普通（DEF 帧 0，默认显示） |
| `1` | pressed | 按下（DEF 帧 1，鼠标点击） |
| `2` | highlight | 高亮（DEF 帧 2，预留） |
| `3` | disabled | 禁用（DEF 帧 3） |
